##  Résumé
Ce projet porte sur la mise en place d’un mécanisme sécurisé d’autorisation de déploiements automatisés à l’aide d’**Ansible** pour la configuration d’un serveur web.  
L’objectif est de démontrer comment l’automatisation permet des déploiements **sécurisés, cohérents et reproductibles**, tout en limitant les actions aux utilisateurs autorisés.

---

##  Objectifs
- Automatiser l’installation et la configuration d’un serveur web
- Contrôler les accès et les droits des utilisateurs
- Réduire les erreurs humaines liées aux déploiements manuels
- Améliorer la sécurité et la traçabilité des opérations

---

## Technologies utilisées
- **Ansible**
- **Ubuntu Linux**
- **SSH**
- **Apache / Nginx**
- YAML

---

## Architecture du système
- Un **nœud de contrôle Ansible**
- Un **serveur web cible**
- Communication sécurisée via **SSH**
- Gestion des privilèges avec **sudo**

---

##  Fonctionnement
1. Installation d’Ansible sur le nœud de contrôle  
2. Création de l’inventaire des serveurs  
3. Écriture des playbooks Ansible  
4. Déploiement automatisé du serveur web  
5. Tests et validation des résultats

##  Résultats
Les tests effectués montrent que le serveur web est correctement installé et configuré.  
Ansible garantit que le système reste toujours dans l’état souhaité, même après plusieurs exécutions.

---

##  Conclusion
Ce projet met en évidence l’importance de l’automatisation et du contrôle des autorisations dans l’administration des systèmes.  
Ansible s’impose comme un outil fiable et efficace pour les déploiements modernes.
